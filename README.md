minishell

readline関数を用いてBash 互換の最小シェルを実装しました。プロンプトと履歴、パス解決、パイプ／各種リダイレクト（< > << >>）、
環境変数と$?の展開、クォート規則（’はリテラル、"は$のみ展開）、Ctrl-C/D/\ の対話挙動を再現。ビルトインは echo/cd/pwd/export/unset/env/exit に対応。
設計は Lexer → Parser(AST) → Expander → Executor の 4 段で疎結合化。シグナルはグローバル1個に限定し、ハンドラから共有状態を触らない方針で競合や UB を回避。
ヒアドキュメントはテンポラリ fd で安全に扱い、履歴更新は行いません。ファイル記述子/プロセス/エラー伝播を体系化し、リークゼロ（readline 由来を除く）を保証。
学びとして、プロセス生成とパイプラインの fd 設計、クォートと展開規則の仕様準拠、
異常系（権限／ディレクトリ実行／PATH 未設定等）で bash と整合する終了コードを返すための分岐網羅を徹底しました。
